# 기획서: AI 기반 영어 단어 시험 채점 서비스 (AI Exam Grader)

## 1. 프로젝트 개요

### 1.1. 프로젝트 목표
학원 선생님들의 업무 효율을 극대화하기 위해, 영어 단어 시험지의 정답지(PDF)와 학생 답안지(PDF)를 업구로드하면 자동으로 채점하고 그 결과를 시각적으로 제공하는 웹 서비스입니다. 복잡한 절차 없이 직관적인 탭 기반 인터페이스를 제공하여 사용성을 높입니다.

### 1.2. 핵심 가치
*   **직관성**: 별도의 학습 없이 바로 사용할 수 있는 단순한 UI
*   **시각화**: 채점 결과가 원본 시험지 위에 직관적인 오버레이로 표시
*   **확장성**: 여러 종류의 시험을 탭으로 구분하여 동시에 관리 가능

---

## 2. 개발 스테이지 전략

### Stage 1: 프론트엔드 중심 (Standalone)
*   **목표**: 서버 없이 브라우저 내에서 핵심 기능을 완결성 있게 체험 가능하도록 구현
*   **특징**:
    *   모든 데이터(시험지 정보, 채점 결과 등)는 브라우저 메모리 또는 로컬 스토리지에서 일시적으로 관리
    *   채점 로직은 Mock-up 함수로 대체하거나 간단한 로직으로 시뮬레이션
    *   UI/UX의 상호작용 및 흐름 완성도 확보에 집중

### Stage 2: 백엔드 통합 (Full Integration)
*   **목표**: 실제 AI 채점 로직 및 데이터 처리를 위한 REST API 연동
*   **특징**:
    *   파일 업로드 및 분석 요청을 실제 서버로 전송
    *   채점 알고리즘 연동 (정답 구조 추출 -> 학생 답안 구조 추출 -> 비교)
    *   대용량 PDF 처리 안정성 확보

---

## 3. 디자인 가이드

### 3.1. 디자인 컨셉
*   **Professional & Clean**: 교육 도구로서의 신뢰감을 주는 깔끔하고 정돈된 디자인
*   **Focus on Content**: 시험지와 채점 표시가 가장 눈에 띄도록 배경과 UI 요소는 미니멀하게 구성
*   **Vibrant Accents**: 채점 결과(정답/오답) 표시는 명확한 색상(Red, Green/Blue)을 사용하여 시인성 극대화

### 3.2. 주요 컬러 팔레트
*   **Background**: White, Light Gray (눈의 피로도 감소)
*   **Primary Action**: Deep Blue (신뢰, 안정)
*   **Correct (정답)**: Green or Blue Circle
*   **Incorrect (오답)**: Red Slash/Check

---

## 4. 사이트맵 및 UI 구조

단일 페이지 웹 애플리케이션(SPA) 구조로, **상단 탭 바**와 **메인 워크스페이스**로 구성됩니다.

### 4.1. 메인 레이아웃
*   **상단 헤더 (Tabs Area)**
    *   진행 중인 시험(Tab) 목록 표시
    *   [+] 새 탭 열기 버튼
*   **메인 컨텐츠 영역**
    *   선택된 탭의 상태에 따라 다른 화면 노출

### 4.2. 화면별 상세 구성

#### A. 빈 탭 (초기 상태) / 정답지 등록 화면
*   **UI 구성**
    *   중앙에 강조된 **'정답지 PDF 업로드'** 영역 (드래그 앤 드롭 지원)
    *   파일 선택 버튼
    *   안내 문구: "채점할 시험의 정답지(PDF)를 먼저 등록해주세요."
*   **핵심 기능**
    *   PDF 파일 업로드 처리
    *   업로드 완료 시 -> '시험지 처리 및 탭 이름 설정' 프로세스 진입
    *   처리 완료 후 -> **B. 채점 워크스페이스**로 자동 전환

#### B. 채점 워크스페이스 (메인)
*   **좌측 패널: 학생 시험지 리스트 & 업로드**
    *   상단: **'학생 시험지 추가'** 버튼 (다중 파일, 드래그 앤 드롭)
    *   리스트 목록:
        *   학생 이름(또는 파일명)
        *   채점 상태 (대기중 / 채점중 / 완료 / 실패)
        *   점수 요약 (예: 18/20)
    *   선택 시 우측 패널에 상세 내용 표시
*   **우측 패널: 채점 결과 뷰어**
    *   **PDF 뷰어 영역**:
        *   학생이 제출한 원본 PDF 표시
        *   각 문항 위치에 채점 마크(O, X) 오버레이
    *   **결과 사이드바 (Toggle 가능)**:
        *   총점 및 정답률
        *   틀린 문제 리스트 (클릭 시 해당 문제 위치로 스크롤 이동)

## 5. 데이터 명세 (Data Spec)

*테크니컬한 타입 정의 대신 데이터의 논리적 구조를 기술합니다.*

### 5.1. 시험(Exam) 세션 데이터
*   **Session ID**: 탭을 구분하는 고유 키
*   **Exam Name**: 정답지 분석 후 추출된 시험 제목 (탭 이름으로 사용)
*   **Answer Key Data**:
    *   원본 PDF 파일
    *   추출된 정답 데이터 (문항 번호 매핑 된 정답 값)
    *   문항별 좌표 정보 (채점 마크 표시 위치)

### 5.2. 학생 답안(Submission) 데이터
*   **Submission ID**: 학생 답안 고유 키
*   **Student Name**: 파일명 등에서 추출한 학생 식별자
*   **Status**: Uploaded -> Analyzing -> Graded
*   **Score Data**:
    *   총 문항 수
    *   맞은 개수
    *   틀린 개수
    *   각 문항별 판독 결과 (학생 답안, 정답 여부)
*   **Overlay Data**:
    *   채점 마크(O/X)가 그려질 PDF 상의 좌표 및 메타데이터

## 6. 기능 명세

### 6.1. 탭 매니지먼트
*   **새 탭 생성**: [+] 버튼 클릭 시 새로운 격리된 작업 공간 생성
*   **상태 보존**: 다른 탭을 다녀와도 기존 작업 내용(업로드된 파일, 채점 현황) 유지
*   **탭 닫기**: [x] 버튼 클릭 시 해당 시험 세션 및 데이터 휘발

### 6.2. 정답지 처리 프로세스
*   사용자가 PDF 업로드
*   시스템이 PDF에서 메타데이터(시험 제목 등) 추출 시도 -> 실패 시 파일명으로 대체
*   정답 구조(문항 배치, 정답 텍스트 등) 분석 및 저장

### 6.3. 학생 시험지 채점 프로세스
*   다수의 PDF 동시 업로드 지원
*   각 파일별 비동기 처리
*   **채점 로직**:
    1.  학생 답안지 구조 분석 (정답지와 동일한 템플릿인지 확인)
    2.  학생 수기 답안 인식 (OCR 등 API 활용 가정)
    3.  정답 데이터와 비교하여 정오답 판별
    4.  결과 데이터 생성

### 6.4. 결과 시각화
*   PDF 렌더링 위에 Canvas 레이어를 겹쳐서 구현
*   정답(O): 파란색/초록색 동그라미
*   오답(X): 빨간색 빗금 또는 체크
*   반응형: PDF 확대/축소 시 마크 위치 동기화
